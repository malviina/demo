First section: Using triggers

This is an example of the use of a trigger to make something happen. I'm using a waypoint.js javascript library to trigger the event. As the trigger below (highlighted with the text "trigger") reaches the halfway of the screen (as it's offset is set to 50%, otherwise the event would be triggered as the trigger reaches the top of the screen) the picture on the right emerges. And as you scroll up again it fades away.

TRIGGER

The fading effect is done with css-properties opacity and transition.

The picture has a position attribute value sticky. That makes it stick to certain position on the screen until the end of the section is reached. There were other ways to achieve the same effect. One was making a div with a height of screen height and making it change it's position from relative to fixed, and keeping the desired figure in the middle of the div. But this seems to work well enough for now.

At the end of this section I've positioned another trigger. That one makes the picture fade out again.

This is a good way to show fact box or maps or figures as the text is scrolled.

Second section: Drawing vector graphics through scroll

Scroll down to draw a triangle.

Scroll back up to reverse the drawing.

This trick I learned from this tutorial by w3schools.com.

However, to make it work on this demo page, I had to tweak it a little. The scrollpercentage calculation had to be fixed, as I want the path to reveal itself as the section is scrolled through instead of the whole page, which is how it's done on the tutorial.

In the tutorial, the percentage of the scroll (scrollpercent) was calculated like this:

       var scrollpercent = (document.body.scrollTop +
            document.documentElement.scrollTop)
            / (document.documentElement.scrollHeight
            - document.documentElement.clientHeight);

This gives the value of how much the body element is scrolled down compared to the height of the whole document.

I created a little "helper" div element that I places at the beginning of the section. It has a certain height, zero width, and it is floating on the side. The scrollpercent is calculated against the helper instead of the whole document. 


        var helper = document.getElementById("triangle-helper");
        var helper_height = helper.scrollHeight;
        var screen_height = document.documentElement.clientHeight;

        var rect = helper.getBoundingClientRect(); // This is to
        find the position of the helper in relation to the 
        window screen
        var helper_scrolled = screen_height - rect.top - 500; 
        // the last value gives some offset so that the drawing
        begins a bit later then when the sections top comes to view

        if (helper_scrolled < 0) {
            helper_scrolled = 0;
        }

        if (helper_scrolled > helper_height) {
            helper_scrolled = helper_height;
        }

        var scrollpercent = helper_scrolled / helper_height;

Section 3: Autoplaying videos

This is something I really like in a good visual digital storytelling. Videos that automatically start playing as the page is scrolled, especially when they are fullscreen. No clicking! With few lines of code the video can be made to play automatically once certain fraction of it is visible.

Section 5: Changing backgrounds

Now this section shows changing backgrounds. I first tried to make this by changing the background-image property of a div that covers the whole screen. That worked, but not on all browsers. Here I'm using just images. First I made the images stick to the screen by changing the position property of the element to fixed, but that resulted the position to lag in Firefox, creating a very non-smooth experience. Thus, I used sticky property again. It doesn't work on all browsers, though, so I need to find a solution to that. Next step would be making a function that resizes the image so that it would cover the whole screen even when the height of the screen was larger than the width.
